PROMPT 2.3 — What-If Analysis Engine
============================================================

PROMPT --- Scenario comparison --- the core feature workers use
most

Context: factory_simulation.py and math_engine.py are complete.

Create: what_if_engine.py

This module runs scenario comparisons for factory managers.

Implement:

Class Scenario:

- name: str

- description: str

- parameter_changes: dict --- what is different from baseline

Example: {\"station_3_num_machines\": 2} or {\"demand\": 250} or
{\"operator_count\": 3}

Function run_what_if(baseline_config, scenarios: list[Scenario],
data_source) -> WhatIfResult:

1. Run baseline simulation (30 reps)

2. For each scenario: clone config, apply parameter_changes, run 30
reps

3. Compare each scenario to baseline

4. Return WhatIfResult

Class WhatIfResult:

- baseline: SimulationResult

- scenarios: list of ScenarioResult

- Method to_summary_dict() -> dict --- for sending to Android as JSON

Class ScenarioResult:

- scenario: Scenario

- result: SimulationResult

- throughput_delta: float --- change from baseline

- throughput_delta_pct: float

- lead_time_delta: float

- recommendation: str --- plain English, 1 sentence, no jargon

Example: \"Adding one machine at Station 3 increases daily output by 23
units (+12%)\"

Pre-built scenarios (create these as ready-to-use defaults):

1. add_machine_at_bottleneck --- automatically finds bottleneck and
adds 1 machine there

2. add_one_operator --- increases operator_count by 1

3. reduce_batch_size_half --- halves current batch_size

4. increase_demand_20pct --- increases demand by 20%

5. reduce_cycle_time_10pct --- reduces all cycle times by 10% (process
improvement scenario)

6. two_shift_operation --- doubles shift_hours
