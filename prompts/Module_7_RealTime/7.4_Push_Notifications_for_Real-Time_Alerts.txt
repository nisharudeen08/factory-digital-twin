============================================================
PROMPT 7.4 — Push Notifications for Real-Time Alerts
============================================================

PROMPT --- Notify factory supervisor instantly when a bottleneck
forms

**Context: Real-time data is flowing from machines to the digital
twin.**

Now I need the system to automatically alert the factory supervisor
when a bottleneck forms ---

without the supervisor having to open the app.

Implement push notifications:

PYTHON SIDE --- alert_engine.py:

- Monitor incoming sensor data in background

- Trigger alert when: utilization of any station exceeds 0.90 for more
than 2 consecutive minutes

- Trigger alert when: queue at any station exceeds 10 jobs

- Trigger alert when: machine goes from \"running\" to \"broken\"

- Use Firebase Cloud Messaging (FCM) to send push notification to
registered Android devices

- Alert payload: { \"station_id\": 3, \"alert_type\": \"bottleneck\",
\"utilization\": 0.94, \"message_en\": \"Bottleneck forming at Welding
Station\", \"message_ta\": \"வெல்டிங் நிலையத்தில் தடை உருவாகிறது\" }

- De-bounce: do not send more than 1 alert per station per 10 minutes

ANDROID SIDE:

- Integrate Firebase SDK (add google-services.json)

- Register device FCM token with Python server on app startup: POST
/register_device

- Handle incoming FCM notification:

- Show as Android notification with factory icon

- Tapping notification opens BottleneckFragment directly

- Notification text in user's selected language (Tamil or English)

- Notification channel: \"Factory Alerts\" --- high importance, bypass
do-not-disturb for critical alerts

SUPERVISOR ROLE:

- In SettingsScreen: toggle \"Receive Alerts\" on/off

- Set alert threshold: \"Alert me when utilization exceeds \_\_\_%\"
(slider: 80-99%)

- Quiet hours: no alerts between 10pm and 6am

Show complete code for both Python FCM integration and Android
notification handling.
